## 🤝 Code of Conduct
 
This project runs on **Proof of Burnout** — not proof of toxicity.
 
We expect all contributors to:
 
- Be respectful and inclusive, even when the meeting goes into hour three
- Give constructive feedback, not performance reviews
- Assume good intent before escalating to HR
- Remember: we're all just trying to survive out here
Unacceptable behavior will be handled swiftly — faster than your manager responds to Slack messages on a Friday afternoon.

## 🐛 Reporting Issues
 
Found something broken? A dead link? A typo that slipped past QA (which is you)?
 
1. Go to [Issues](https://github.com/CubertCoin/cubertcoin/issues)
2. Check if the issue already exists — we don't do duplicate work here (unlike some departments)
3. If not, open a new issue using the appropriate template:
   - 🐛 [Report a bug](https://github.com/CubertCoin/cubertcoin/issues/3)
   - 🎨 [Submit content](https://github.com/CubertCoin/cubertcoin/issues/1)
**When reporting a bug, please include:**
 
- A clear title (not "it's broken" — we need more than that)
- What you expected to happen
- What actually happened
- Steps to reproduce

## 🎨 Submitting Content
 
Content is the lifeblood of the Corporate Survivor community. We accept:
 
- **Memes** — must be relatable to office/corporate life. No gratuitous cruelty, just shared pain.
- **Written content** — blog-style posts, Twitter/X threads, LinkedIn takes, survival guides
- **Illustrations & design** — must align with the [Brand Book](https://github.com/CubertCoin/cubertcoin/blob/main/brand-book)
- **Video/audio** — short-form content about corporate life
### Content Guidelines
 
- Keep it relatable — the humor comes from shared experience, not mockery of individuals
- Avoid targeting real companies or specific people
- Stay consistent with the CubertCoin tone: dry, sardonic, but ultimately hopeful
- Original content only — no plagiarism (we have a zero-tolerance policy, unlike our manager)
To submit, open a [content submission issue](https://github.com/CubertCoin/cubertcoin/issues/1) with a description and link or attachment.

## ✏️ Style Guide
 
When contributing text to the repository, follow the CubertCoin voice:
 
- **Tone:** Dry, self-aware, relatable. Think office humor, not roast comedy.
- **Perspective:** First person plural ("we", "our") when speaking as the project
- **Formatting:** Use Markdown. Headers, tables, and bullet points are encouraged — we love a good status update
- **Emoji:** Used sparingly for section headers and lists. Not in every sentence. We're professionals. Sort of.
- **Jargon:** Corporate buzzwords can be used ironically (e.g., "synergize", "circle back") — but always with a wink

## 🔀 Pull Request Process
 
For documentation or repository changes:
 
1. **Fork** the repository
2. Create a branch with a descriptive name:
   ```
   git checkout -b fix/broken-link-readme
   git checkout -b content/meme-monday-batch-1
   ```
3. Make your changes
4. Commit with a clear message:
   ```
   git commit -m "fix: update Discord invite link in README"
   git commit -m "content: add Q3 performance review meme"
   ```
5. Open a Pull Request against `main` with a description of what you changed and why
6. Wait for review — we promise to respond faster than your skip-level
**PR Checklist:**
 
- [ ] Changes are scoped and focused (one thing per PR)
- [ ] Content aligns with the Brand Book and tone guide
- [ ] No broken links
- [ ] Spelling and grammar checked (we're not animals)
